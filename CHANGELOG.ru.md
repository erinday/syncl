# Список изменений

Все значимые изменения в этом проекте будут документироваться в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и этот проект следует [Семантическому Версионированию](https://semver.org/spec/v2.0.0.html).

---

## [2.0.0] - 2026-04-22

### ⚠️ Критические изменения

- **Изменён формат имени события**: с `{namespace}_update` на `{namespace}:update`
  - Было: `__ls_update` (с namespace по умолчанию `__ls_`)
  - Стало: `__ls:update` (с namespace по умолчанию `__ls`)

### Добавлено

- Новое свойство-геттер `eventUpdateName` для получения имени события, которое вызывается при изменениях в storage (в текущей вкладке)
- Поддержка SSR (Server-Side Rendering) - библиотека теперь безопасно проверяет существование `window`
- Поле `prefix` для внутреннего управления ключами (отделено от логического `namespace`)

### Изменено

- **Namespace по умолчанию изменён**: с `'__ls_'` на `'__ls'` (без нижнего подчёркивания в конце)
- Улучшен метод `clean()`:
  - Теперь сохраняет ключ версии при очистке
  - Отправляет событие обновления только если что-то реально изменилось
- Лучшая изоляция между логическим namespace (для событий) и префиксом storage (для ключей)

### Руководство по миграции

#### До (v1.x)
```ts
const store = new Syncl({
  namespace: '__app_'  // с нижним подчёркиванием в конце
})

// Имя события: '__app_update'
// Имя ключей: '__app_KEY'
```

#### После (v2.x)
```ts
const store = new Syncl({
  namespace: '__app'  // без нижнего подчёркивания в конце
})

// Имя события: '__app:update'
// Имя ключей: '__app_KEY'  // не изменилось
console.log(store.eventUpdateName) // '__app:update'
```

## [1.0.1] - 2026-04-19

### Добавлено

- Первый публичный релиз
- Синхронизация между вкладками через нативное событие `storage`
- Реактивность внутри вкладки через пользовательские события
- Поддержка `localStorage` и `sessionStorage`
- Изоляция через namespace для предотвращения конфликтов ключей
- Инвалидация хранилища на основе версии
- Ноль внешних зависимостей
- Поддержка TypeScript с generic типами для ключей

### Методы

- `getValue(key)` - получить значение по ключу
- `setValue(key, value)` - установить значение с автоматическим определением изменений
- `removeValue(key)` - удалить значение и вызвать событие обновления
- `clean()` - очистить все данные в namespace
- `on(cb)` - подписаться на изменения в хранилище
- `emit()` - вручную вызвать событие обновления

### Параметры

- `version` - версия хранилища (по умолчанию: '1')
- `namespace` - префикс для всех ключей (по умолчанию: '__ls_')
- `storage` - экземпляр хранилища (по умолчанию: localStorage)
- `eventName` - имя пользовательского события для обновлений внутри вкладки (по умолчанию: '__ls_update')
